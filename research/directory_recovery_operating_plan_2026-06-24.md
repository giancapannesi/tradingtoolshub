# TradingToolsHub Directory Recovery Operating Plan

Date: 2026-06-24

## Objective

Make TradingToolsHub work as a live trading-tools directory, not a static content dump.

The directory should help traders compare tools, prop firms, brokers, data APIs, charting platforms, options-flow products, and trading journals. The business goal is to recover Google visibility, increase indexed money pages, and route search demand into review, alternatives, comparison, and best-of pages with affiliate or lead value.

## Operating Principles

1. Static queues are not strategy.
2. Daily targets come from live GSC, indexation state, current SERPs, freshness gaps, affiliate value, and internal-link gaps.
3. Queue files are short execution buffers only.
4. OpenAI/Codex chooses and creates the work where possible.
5. Every generated piece must support a directory page, not exist as isolated blog content.
6. Recovery is measured by indexed pages, impressions, clicks, CTR, average position, and affiliate-intent page growth.

## Content Architecture

Each cluster needs four page types:

1. Hub page
   - Example: `/best/best-prop-firms/`
   - Purpose: rank for broad commercial query.

2. Review pages
   - Example: `/review/polygon-io/`
   - Purpose: tool-specific intent and affiliate conversion.

3. Comparison pages
   - Example: `/compare/polygon-io-vs-alpha-vantage/`
   - Purpose: decision-stage search.

4. Support articles
   - Example: `/blog/polygonio-pricing-guide-2026-all-plans-costs-hidden-fees/`
   - Purpose: long-tail search and internal links into reviews/hubs.

No support article should publish without links into the directory layer.

## What To Remove Or Replace

Remove as strategy:
- Long static `blog_queue.json` strategy behavior.
- Empty `content_queue.json` as a fake active engine.

Keep only as execution buffers:
- A daily generated `today_queue.json` or equivalent short list.

Replace with:
- `tth_dynamic_content_selector.py`
  - Reads GSC, indexation, site inventory, competitors, logs.
  - Scores targets.
  - Writes only today’s tasks.
  - Fails if it cannot produce a justified target list.
