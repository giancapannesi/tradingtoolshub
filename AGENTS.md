## Safety Rules

- Do not delete or rewrite prop-firm work without explicit founder approval.
- Do not guess affiliate URLs. Leave them empty unless verified.
- Check existing tool JSON schema before adding profiles.
- Expect large generated content batches. Preserve existing local changes.

## Dynamic Content Rule

- Do not treat static queues as the source of truth for TradingToolsHub content.
- Use live signals first: GSC queries/pages, indexation state, traffic logs,
  affiliate/business priority, freshness age, and internal-link gaps.
- Static queue files are allowed only as short execution buffers for the next
  concrete run, not as months-long autonomous strategy.
- When content must be generated by API, use the most capable available OpenAI
  model for the task by default, with explicit environment override allowed.
- Do not expand new Claude/Gemini/other-LLM dependencies for TTH content
  generation when an OpenAI model can do the work.
- Existing non-OpenAI content generators should be considered legacy and
  replaced or wrapped by a live OpenAI/Codex-driven selector before further
  strategic expansion.
